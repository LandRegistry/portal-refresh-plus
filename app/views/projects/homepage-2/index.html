Designs

The current Homepage has colours and styles not consistant with GOV Design or the new HMLR Design System. It also recieved negative feedback from users saying it's not easy to find the services they need and that it is too much to scroll. There is positive feedback on the shortcuts and requests for more of them as this helps them navigate to the services they need.

These early designs aimed to address some of these problems without changing the content of the page but remain unfinished with the more finished versions existing in the figma. The main use of these current prototypes is the coded header and how we might iterate to an ideal one based on the HMLR design system. The code wasn't easy to work with as there was a lot of styling and layout code to un-do. In the future when the new header/homepage gets built it might be worth creating a fresh Heroku prototype.

  • Design A - First iteration where the header moves across to the HMLR look but keeps the same navigation in header. Navigation links now triggered on click instead of on hover (nested dropdowns not yet coded). Drop down colours taken from GOV.UK. Shortcut styling also changed to a more GOV feel. The rest of the homepage remains the same.
  • Design B - This design tried out blocking out the navigation dropdown to the full width of the screen similar to GOV.UK but you can see the list of services are just too long to make this design work.
