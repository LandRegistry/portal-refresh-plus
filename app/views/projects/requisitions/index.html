Requisitions

Overview

There has been user research and design in the past related to surfacing requisitions/informing the user that they need to require action, but this work is bypassing that user research and design, due to the urgency of implementation from the Product Team.

Project team

  • Andy Carr - Interaction design
  • John Taylor - Product owner
  • Pat Gannon - Business analyst
  • Gareth Roberts - Product manager

Design

E23-66i - Requisition issued visible from collapsed row

Users need to be able to tell that a requisition has been issued, without having to expand application rows.

  • Design option 1 - The expand/collapse icon has been changed to a warning sign. This has previously tested successful with users.
  • Design option 2 - The expand/collapse icon has been changed to the letter R, which stands for requisition. My criticism of using the R is that it will require further explanation, where as the icon already tested well without explaination.
  • Design option 3 - Keeps the expand/close icon and adds an R. My criticism of using the R is that it will require further explanation, where as the icon already tested well without explaination.
